## User Request

Create comprehensive IVR Integration Architecture documentation covering the Integration Services Layer (ISL) and its interaction with IVR consumers (Genesys, Cognigy, VA).

### Background

What's referred to as "Core Platform" in many documents shall be called "Integration Services Layer (ISL)". It's not only an IVR backend - it's used to:
- Cache policy and claim information required for CSA implementation of GCP APIs
- Serve as integration services with eClaims platform
- Support IVR interaction via policy-service APIs
- Aggregate events from backend policy and claim management systems

The reason events are sent in the first place: backend system performance/latency, reliability, availability, and load tolerance qualities are inadequate to support low latency, high availability requirements of IVR and similar systems.

### Deliverables Required

**Two main documentation themes:**
1. **API Sequences** - Concentrates on IVR API request flow from Genesys → Cognigy/VA → ISL
2. **IVR Data Flow** - End user through Genesys to ISL and backends

**Three audience levels for each:**
1. **L1:** Rich in tech details for technical audience
2. **L2:** Simplified for BA/PM/Technical Management  
3. **L3:** Principal aspects suitable for leadership team
