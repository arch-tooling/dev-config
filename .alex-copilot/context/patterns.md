# Work Patterns and Reminders

Personal productivity patterns. Reference when starting work or wrapping up.

## Communication

- **Practical vs possible**: Before going deep on a topic, ask "Is this actionable this quarter?" If not, park it. This is the single most-repeated theme in my goals (6 mentions across Q2-Q4).
- **Structured updates**: When reporting status to Daniel or stakeholders, lead with what's done, then what's next, then blockers. No preamble.
- **Paraphrase before responding**: In meetings, restate what was asked before answering. Prevents misalignment.
- **Avoid filler**: Cut "basically," "essentially," "so," "like" from verbal delivery. Focus on concise, structured responses.

## Task Management

- **Log unplanned work**: When pulled into something unplanned, note it immediately. This feeds commercial tracking evidence and weekly Confluence updates.
- **One thing at a time**: "What did I finish before starting something else?" -- track completion, not just activity.
- **Weekly capture**: Every Friday (or Monday AM), update the Confluence weekly view. Do not let it slip past Monday.

## Meeting Prep

- Before any meeting with Daniel: check commitments.md for anything due or overdue.
- Before architecture meetings: check current ARCHI status and any open action items.
- Before stakeholder meetings: prepare 2-3 bullet summary of current state, no more.

## Documentation

- Keep documents in UTF-8 markdown. Use Mermaid for diagrams.
- For formal delivery: run the document generation pipeline (markdown to Word/PPTX).
- Fact-check before sending. Use verification tags on uncertain claims.
