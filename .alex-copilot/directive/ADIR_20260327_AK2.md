## Known Limitations & Risk Mitigation

### Session Collision Risk

**Issue**: Single user working across multiple projects simultaneously could create conflicting instructions

**Scenarios**:
- Alex starts task in gga-standards, session open
- Alex starts second task in VA, different session
- Both sessions receive conflicting instructions
- Both try to update same PROMPT/DONE files

**Mitigation** (Timestamp-based):
- PROMPT/DONE files use timestamp in filename → unlikely collision within same minute
- Project scoping prevents cross-project file conflicts
- File system timestamps show last-modified → can detect overlaps
- If collision suspected: Check file timestamps, read both files, resolve manually

**Acceptance**: Alex accepts this limitation until better solution identified

**Monitoring Trigger**: If >2 collisions per week, revisit architecture

---

### Detection Mechanism (Future Enhancement)

**Proposal** (not in AK: 2 scope):
1. Add session ID to PROMPT/DONE files
2. Lock file mechanism in alex/.copilot-task-state/
3. Session registry: alex/.copilot-task-state/ACTIVE_SESSIONS.json
4. Copilot checks for concurrent sessions before file operations
